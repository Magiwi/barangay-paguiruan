# User Manual

## e-Governance System for Barangay Paguiruan

Ang dokumentong ito ay gabay para sa mga gumagamit ng e-Governance System ng Barangay Paguiruan. Saklaw nito ang pangunahing gamit ng system para sa `Resident`, `Staff`, at `Admin`.

## 1. Layunin ng System

Ang e-Governance System ay ginawa upang mapadali ang mga sumusunod:

- Online resident registration
- Secure login at account management
- Request ng barangay documents
- Online permit application
- Pagsusumite ng issue o complaint
- Request ng blotter record
- Pagtingin ng announcements at notifications
- Management ng registrations, residents, reports, at operational records para sa staff at admin

## 2. Mga Uri ng User

May tatlong pangunahing uri ng user sa system:

- `Resident`
  Gumagamit para mag-register, mag-request ng documents, permits, blotter records, at magsumite ng issues.
- `Staff`
  Gumagamit para sa registration review, blotter operations, announcements, complaints, at reports depende sa assigned access.
- `Admin`
  May mas malawak na control sa residents, registrations, officials, puroks, certificates, permits, complaints, reports, audit, at SMS.

## 3. Paano Mag-access ng System

1. Buksan ang homepage ng system.
2. I-click ang `Log In` kung may account na.
3. I-click ang `Register` kung wala pang account.

## 4. Registration ng Resident

### 4.1 Mga Hakbang sa Pag-register

1. Sa homepage, i-click ang `Register`.
2. Kumpletuhin ang form:
- Personal Information
- Address Information
- Contact and Demographics
- Classification details kung applicable
- Account Information
3. Mag-upload ng hinihinging proof o ID kung kailangan.
4. Basahin at i-confirm ang data privacy consent.
5. I-click ang `Register` o submit button.

### 4.2 Mahahalagang Paalala

- Dapat valid at unique ang email address.
- Ang password ay kailangang tumugma sa confirmation field.
- Ang registration ay maaaring `pending approval` muna bago makapag-login.
- Kung suspended ang account o hindi pa approved, hindi makakapasok sa dashboard.

## 5. Login at Logout

### 5.1 Login

1. I-click ang `Log In`.
2. Ilagay ang registered email.
3. Ilagay ang password.
4. Optional: i-check ang `Remember me`.
5. I-click ang `Sign In`.

### 5.2 Kapag Nakalimutan ang Password

1. Sa login page, i-click ang `Forgot your password?`
2. Ilagay ang registered email address.
3. Sundin ang reset instructions na ipapadala sa email kung configured ang mail system.

### 5.3 Logout

1. I-click ang user menu o account menu.
2. Piliin ang `Sign Out`.

## 6. Resident User Guide

Kapag approved na ang resident account at naka-login, mapupunta ang user sa `Resident Dashboard`.

### 6.1 Main Navigation ng Resident

Makikita sa resident panel ang mga sumusunod:

- `Home`
- `About`
- `Officials`
- `Services`
- `Announcements`
- `Notifications`
- `Profile Settings`
- `Change Password`

### 6.2 Request Documents

Para mag-request ng barangay document:

1. Buksan ang `Services`.
2. Piliin ang `Request Documents`.
3. I-click ang paggawa ng bagong request.
4. Piliin ang document type.
5. Punan ang hinihinging impormasyon.
6. I-submit ang request.
7. Bantayan ang status hanggang ma-release.

Karaniwang mga status:

- `Pending`
- `Approved`
- `Rejected`
- `Released`

### 6.3 Apply for Permit

1. Buksan ang `Services`.
2. Piliin ang `Apply for Permit`.
3. Gumawa ng bagong permit request.
4. Punan ang permit details.
5. Mag-upload ng supporting documents kung kinakailangan.
6. I-submit ang application.
7. I-monitor ang status sa permit list.

### 6.4 Report Issue

1. Buksan ang `Services`.
2. Piliin ang `Report Issue`.
3. I-click ang pag-create ng bagong report.
4. Ilagay ang title, detalye, at iba pang hinihingi.
5. I-submit ang issue report.
6. Balikan ang issue list para sa updates at status.

### 6.5 Request Blotter Record

1. Buksan ang `Services`.
2. Piliin ang `Request Blotter`.
3. I-click ang bagong request.
4. Ilagay ang blotter details o request details.
5. I-submit ang request.
6. Tingnan ang status sa `My Blotter Requests`.

### 6.6 Announcements

1. I-click ang `Announcements`.
2. Tingnan ang listahan ng approved announcements.
3. I-click ang announcement para sa buong detalye.

### 6.7 Notifications

1. I-click ang bell icon o ang `Notifications`.
2. Tingnan ang bagong updates.
3. I-click ang notification para buksan ang related page.
4. Puwedeng gamitin ang `Mark all read` kung kailangan.

### 6.8 Profile Settings

Sa `Profile Settings`, maaaring:

- I-update ang personal profile details
- Mag-manage ng family members kung head of family
- Mag-submit ng verification updates depende sa feature availability
- Mag-request ng household head transfer kung applicable

### 6.9 Change Password

1. Pumunta sa `Change Password`.
2. Ilagay ang current password.
3. Ilagay ang bagong password.
4. I-confirm ang bagong password.
5. I-save ang changes.

## 7. Staff User Guide

Ang staff panel ay depende sa permissions na ibinigay ng admin. Hindi lahat ng staff ay pare-pareho ang makikitang modules.

### 7.1 Main Modules ng Staff

Maaaring makakita ang staff ng mga sumusunod:

- `Dashboard`
- `Registration Mgmt`
- `e-Blotter`
- `Announcements`
- `Issue Reports`
- `Reports & Analytics`

### 7.2 Registration Management

Para sa pag-review ng resident registrations:

1. Buksan ang `Registration Mgmt`.
2. Tingnan ang `Pending` registrations.
3. Suriin ang detalye ng user at attachments.
4. Piliin ang tamang action:
- `Approve`
- `Reject`
- `Suspend`
- `Unsuspend`
5. Tingnan ang `Approval History` para sa past actions.
6. Tingnan ang `Verifications` para sa PWD at Senior verification requests.

### 7.3 e-Blotter

Sa `e-Blotter`, maaaring:

- Mag-create ng bagong blotter record
- Mag-edit ng existing record
- Mag-view o download ng evidence
- Mag-manage ng summons
- Mag-manage ng hearings
- Mag-process ng resident blotter requests

### 7.4 Announcements

1. Buksan ang `Announcements`.
2. Gumawa ng bagong announcement kung may access.
3. I-edit ang existing announcement kung kinakailangan.
4. Depende sa workflow, maaaring may approval or review step.

### 7.5 Issue Reports

Sa complaints o issue reports module, maaaring:

- Tingnan ang listahan ng reports
- Buksan ang detalye ng issue
- Magdagdag ng notes
- Mag-assign ng issue sa sarili o sa iba
- I-update ang status depende sa process

### 7.6 Reports and Analytics

Sa reports module, maaaring makita ang:

- Overview
- Population
- Classification
- Services
- Households
- Blotter reports

May export options depende sa page, gaya ng:

- `PDF`
- `CSV`
- `Excel`

## 9. Common Status Meanings

Narito ang ilan sa mga karaniwang status na puwedeng makita:

- `Pending`
  Nakasubmit na ngunit hindi pa naa-actionan.
- `Approved`
  Na-review at tinanggap.
- `Rejected`
  Hindi tinanggap ang request o registration.
- `Released`
  Tapos na at available na ang requested document.
- `Suspended`
  Pansamantalang hindi pinapayagang mag-access ang account.
- `Open`
  Aktibong concern o issue.
- `Resolved`
  Naayos na ang concern.
- `Closed`
  Tapos na ang case at isinara na.

## 10. Troubleshooting

### 10.1 Hindi Maka-login

Mga dapat i-check:

- Tama ang email at password
- Approved na ang account
- Hindi suspended ang account
- Gumagana ang internet connection at server

### 10.2 Walang Lumalabas na Module sa Staff

Posibleng dahilan:

- Wala pang assigned module access
- Naalis ang permission ng account
- Ibang role ang naka-login

### 10.3 Hindi Magsubmit ang Form

Mga dapat i-check:

- Kumpleto ang required fields
- Tama ang file type ng uploads
- Hindi lampas sa allowed file size
- Valid ang email at contact details

### 10.4 Walang Natatanggap na Password Reset Email

Posibleng dahilan:

- Mali ang email address
- Hindi pa configured ang mail service
- Napunta sa spam folder

## 11. Best Practices sa Paggamit

- Gumamit ng totoong impormasyon sa registration.
- Huwag ibigay ang password sa ibang tao.
- Siguraduhing tama ang uploads bago magsubmit.
- I-review ang entries bago i-save o i-submit.
- Mag-sign out lalo na kung shared ang computer.

## 12. Contact and Support

Kung may problema sa paggamit ng system, makipag-ugnayan sa barangay office o sa system administrator para sa:

- account concerns
- approval delays
- access problems
- record correction requests
